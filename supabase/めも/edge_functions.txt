edge_functions役割推測

 メール関連

  - send-contact-email - お問い合わせフォームからのメール送信
  - send-signup-email - ユーザー登録時の確認メール送信

  🤖 AI・Claude関連

  - claude-proxy - ClaudeAI APIへのプロキシ（API キー隠蔽）
  - test-claude - Claude API接続テスト
  - analyze-content - 作品内容の AI分析
  - work-claude-batch - 作品データの一括AI処理

  🔄 バッチ処理系

  - batch-processing - 汎用バッチ処理エンドポイント
  - user-creation-batch - ユーザー一括作成（テストデータ？）
  - author-generation-batch - 著者データ一括生成
  - content-scraping-batch - 外部コンテンツスクレイピング

  📁 データ管理

  - storage-upload - ファイルアップロード処理
  - user-management - ユーザー管理操作
  - work-management - 作品管理操作

  🎯 推薦システム

  - get-recommendations - 推薦データ取得
  - recommendations - 推薦処理（重複？）
  - generate-embeddings - ベクトル埋め込み生成
  - calculate-user-preferences - ユーザー嗜好計算
  - calculate-score - スコア計算（推薦用？）

  ⚙️ 管理・テスト

  - promote_admin - 管理者権限付与
  - init_admin - 管理者初期化
  - test-env - 環境変数テスト
  - simple-test - 動作確認用

  🚨 無駄そうな関数

  重複の可能性

  - get-recommendations と recommendations - 機能が重複？
  - test-claude と simple-test - テスト系が複数

  開発時の残骸

  - user-creation-batch - テストデータ作成用？
  - author-generation-batch - 開発初期の大量データ生成？
  - content-scraping-batch - 初期データ収集用？

  一回限りの処理

  - init_admin - 初回セットアップ用（削除可能）
  - promote_admin - 手動管理用（頻度低い）

  💡 最適化提案

  統合可能

  user-management + work-management → content-management
  get-recommendations + recommendations → unified-recommendations
  test-* 系 → single-test-suite

  削除候補

  - 使用頻度の低いバッチ処理系
  - 重複テスト関数
  - 一回限りの初期化関数
